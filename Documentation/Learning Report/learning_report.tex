\section{Reflection on used methodology}
Reflecting on my workflow so far, using Scrum has been the right choice for building this demonstrator. 
Since the setup relies heavily on quantum interference, the technical requirements often shift as I learn more about how the physical hardware behaves. 
The iterative nature of Scrum has been incredibly useful here. 
Working in weekly sprints allows me to tackle the project in manageable pieces, giving me dedicated time to sit down, evaluate the current problems, and adjust our direction.
This flexible approach means the project isn't forced to stick to a rigid first draft, instead, I remain free to adapt to new approaches and insights as the experiment evolves.
